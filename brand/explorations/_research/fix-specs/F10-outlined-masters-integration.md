# F10 — Architecture: spec the generator + outlined-masters integration (the durable core)

## Root cause

Every shipped SVG is hand-authored, and the wordmark block

```svg
<g font-family="Cinzel, Marcellus, 'Trajan Pro', serif">
  <text x="80" y="160" font-size="120" font-weight="500" letter-spacing="12" fill="...">RTAM</text>
  <text x="540" y="160" font-size="62"  font-weight="400" letter-spacing="8"  fill="...">Foundation</text>
</g>
<circle cx="118" cy="188" r="10" fill="..."/>
```

is **copy-pasted verbatim across 9 files** (5 logos + 3 bilingual/donation lockups carry the full RTAM+Foundation+bindu block; the icons/favicon/temple-lockup repeat the bare-R+bindu or anchored-text idiom). There is no single source of truth for any brand constant. A consequence: the bindu's `cx`, the R's coordinates, the colour hexes, the font stack, and the clear-space rule each exist in N independent copies. The June-2026 audit's 15 defects are overwhelmingly **N-place edits waiting to drift** — changing the bindu center (F1/F3) touches ~9 SVGs plus the specimen/mockup HTML and the PNG/PDF exports, ~60 edit sites for one conceptual change. The same copy-paste is why defects like an off-center bindu or a wrong fill can exist in some copies and not others.

The durable fix is not "edit all copies correctly once" — it is to **make the copies derived artifacts**. One spec file (`brand/spec/brand.json`) holds every brand constant once; a jinja2 generator (`brand/tools/generate.py`) emits all 21 SVGs as live-text masters; an outliner (`brand/tools/outline.py`) emits font-independent path masters via the **already-proven** uharfbuzz→fontTools pipeline; a `build.sh` orchestrates generate → outline → render → gate → measure → contrast so the render-fidelity gate (spike `structural_diff` max-blob ≤150) blocks any regression before commit. After this, F1/F3/etc. are one-line edits to `brand.json`.

This finding owns the **architecture and the integration contract** — the schema, the template structure, the `outline.py` API, the build stage order, the sources-vs-artifacts boundary, the exports policy, and the migration order. The other F-findings supply the *values* that go into the spec (e.g. F1/F3 the bindu grammar, F-color the tokens); F10 supplies the *machine* that consumes them.

## Files touched

This is a research spec; it lists what the **US-13 implementation** will create or change. Nothing is modified in this phase.

New source files (committed, hand-authored):

- `brand/spec/brand.json` — the single source of truth (schema below).
- `brand/spec/brand.schema.json` — JSON Schema validating `brand.json` (build fails closed if spec is malformed).
- `brand/tools/generate.py` — jinja2 renderer: `brand.json` + `templates/*.svg.j2` → live-text SVGs.
- `brand/tools/outline.py` — uharfbuzz→fontTools outliner: live-text SVG (or spec entry) → path-only SVG. API below; lifts `shape()` / `outline_svg()` / letter-spacing semantics directly from `brand/explorations/_research/spikes/shaping_spike.py`.
- `brand/tools/templates/*.svg.j2` — one template per *layout family* (not per asset): `wordmark.svg.j2`, `monogram.svg.j2`, `monogram-circle.svg.j2`, `lockup-stacked.svg.j2`, `lockup-centered.svg.j2`, `seal.svg.j2`, `seal-pattern.svg.j2`, `favicon.svg.j2`. (8 templates cover all 21 assets.)
- `brand/tools/macros.svg.j2` — shared jinja2 macros: `bindu()`, `font_imports()`, `font_faces_local()`, `ground_rect()`.
- `build.sh` — pipeline orchestrator (stage order below).

Generated artifacts (committed, but **never hand-edited** — regenerated by `build.sh`):

- All 21 SVGs under `logos/ icons/ lockups/ seal/` — now `generate.py` output.
- `brand/dist/outlined/*.svg` — path-only distribution masters from `outline.py` (new dir).
- `brand/exports/png/*.png`, `brand/exports/pdf/*.pdf`, `brand/exports/mockups/*.png` — re-rendered downstream (see exports policy).
- `brand/previews/*.html`, `brand/previews/mockups/*.html` — regenerated to reference generated SVGs (these already exist as hand-authored HTML; US-13 makes them templates too, **or** leaves them as-is and only re-exports their PNGs — see Residual risk).

Existing source files that **stop being hand-edited** and become generator inputs: `brand/palette/colors.json` (tokens are *imported into / referenced by* `brand.json`; see schema note — not duplicated).

### 3. Template structure (`templates/*.svg.j2`)

Templates are thin; all numbers come from the asset row + resolved `type_styles`. Shared idioms live in `macros.svg.j2`:

```jinja

{% macro font_imports(families) -%}
<defs><style type="text/css">@import url('https://fonts.googleapis.com/css2?{% for f in families %}family={{ f.import }}{% if not loop.last %}&amp;{% endif %}{% endfor %}&amp;display=swap');</style></defs>

{% macro ground_rect(asset, vb) -%}
{% if asset.ground %}<rect x="0" y="0" width="{{ vb.w }}" height="{{ vb.h }}" fill="{{ tokens[asset.ground] }}"/>{% endif %}
{%- endmacro %}

{% macro bindu(b, ink_cx, baseline, size) -%}
{#  ink_cx  = computed ink-center of the signed glyph (ink_center_x(), baked literal here)
    baseline = the TEXT baseline y of the signed run (NOT the bindu's own cy)
    cy = baseline + cy_offset_ratio*size ; ratios fall back to bindu_spec defaults,
         but the monogram family overrides them per-asset (b.cy_offset_ratio / b.r_ratio). #}
{% if b and b.show %}
{%- set cx  = b.cx if b.cx is defined else ink_cx -%}
{%- set cyr = b.cy_offset_ratio if b.cy_offset_ratio is defined else bindu_spec.cy_offset_ratio -%}
{%- set rr  = b.r_ratio if b.r_ratio is defined else bindu_spec.r_ratio -%}
<circle cx="{{ cx }}" cy="{{ baseline + cyr * size }}" r="{{ rr * size }}" fill="{{ tokens[b.fill] }}"/>
{%- endif %}
{%- endmacro %}
```

### 4. `outline.py` API

```python
# brand/tools/outline.py
def outline_text(ttf: Path, text: str, *, size: float, x: float, y: float,
                 letter_spacing: float = 0.0,
                 anchor: str = "start") -> OutlineResult:
    """Shape `text` with uharfbuzz, extract positioned glyph outlines with
    fontTools, return path data. LIFTED VERBATIM from spikes/shaping_spike.py
    outline_svg(): scale s = size/upem; TransformPen (s,0,0,-s, x+xoff*s, y-yoff*s);
    letter_spacing added once per HarfBuzz CLUSTER boundary incl. the last
    (SVG/CSS semantics); skip empty (space) path data.

    anchor handling (spike README §"text-anchor", to be exercised here):
      shape once to get total advance `adv`; then
        start  -> x as given
        middle -> x_start = x - adv/2
        end    -> x_start = x - adv
      i.e. shift the pen origin, never the per-glyph offsets.
    """

class OutlineResult(NamedTuple):
    paths: list[str]      # SVG `d` strings, one per drawn glyph
    glyph_names: list[str]
    advance: float        # total advance in viewBox units (= end_x - x_start)
    ink_bbox: tuple       # (x0,y0,x1,y1) union of glyph ink, viewBox units


def ink_center_x(ttf, glyph: str, *, size, x, weight, anchor="start") -> float:
    """The bindu cx (F1/F3): horizontal center of the rendered glyph's INK,
    not its advance box. Computes ink_bbox of `glyph` via outline_text and
    returns (x0+x1)/2. This is the ONE place ink-center is computed; both
    generate.py (to bake cx into live-text masters) and outline.py (path
    masters) call it, so the live-text and outlined masters agree by
    construction. Replaces the hand-guessed cx=118 (advance center) with the
    measured ink center (bindu_abc.py found ~126.5 for fs=120)."""


def outline_svg_master(asset: dict, spec: dict, live_svg: Path) -> str:
    """Top-level: given an asset row + resolved spec + its generated live-text
    SVG, emit a PATH-ONLY SVG string. Walks asset.lines, calls outline_text per
    run, replaces each <text> with its <path> elements (same fill), and copies
    through every non-text node (<circle> bindu, <line> rule, <rect> ground,
    <pattern>) UNCHANGED. title/desc/aria preserved. Devanagari + Latin both go
    through outline_text (Tiro and Cinzel TTFs from spec.fonts[*].ttf)."""
```

Key API facts (all pre-proven in spikes, cited so US-13 implements blind):

- Pipeline = `hb.shape` → `tt.getGlyphSet()` → `SVGPathPen` + `TransformPen(s,0,0,-s,x,y)`, `s=size/upem`. (shaping_spike.py:97–127.)
- Letter-spacing: add once per cluster, **including the last** — matches browser SVG `<text>`. Spikes confirmed cross-renderer max-blob 11px on the exact wordmark with ls=12/ls=8.
- `text-anchor=middle/end` (icons, lockups, temple): the spikes did **not** exercise this (README §"Facts unlocked"). US-13 must implement the shift-origin approach above and gate it; this is the one new code path. Latin `R` and Devanagari ऋ monograms are single-cluster so the shift is exactly `−advance/2`.
- The bindu/ring/rule/pattern are already resolution-independent SVG primitives — `outline.py` copies them through untouched. Only `<text>` is replaced.
- Cinzel lacks U+1E5A / U+0323, so the drawn `<circle>` bindu is mandatory and correct (README §last bullet); never expect the font to provide the dot.

### 5. `build.sh` stage order

```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")"                        # repo root

# 0. VALIDATE spec & sync tokens (fail-closed before any render)
python brand/tools/validate_spec.py brand/spec/brand.json brand/spec/brand.schema.json
#    asserts brand.json tokens == palette/colors.json hexes (Rule: one source)

# 1. GENERATE live-text masters  (brand.json + templates -> 21 SVGs in place)
python brand/tools/generate.py --spec brand/spec/brand.json --out brand/

# 2. OUTLINE distribution masters (assets with outlined:true -> dist/outlined/)
python brand/tools/outline.py   --spec brand/spec/brand.json --out brand/dist/outlined/

# 3. RENDER both sets faithfully (Chromium, document.fonts.ready + settle)
python brand/tools/measure_render.py        # live-text masters -> _scratch renders + geometry.json

# 4. GATE: render-fidelity. For every outlined master, structural_diff
#    cairosvg-vs-Chromium max-blob <= 150 (renderer independence), AND
#    outlined-vs-livetext Chromium max-blob <= 150 (the outline equals the
#    live master it was derived from). Calibrated by spikes/negative_control.py.
python brand/tools/render_diff.py --gate dist/outlined   # nonzero exit fails build

# 5. MEASURE: geometry findings (bindu ink-centering, overflow, padding)
python brand/tools/analyze_geometry.py brand/tools/_scratch/geometry.json

# 6. CONTRAST: WCAG check of token pairs actually used
python brand/tools/contrast.py

# 7. EXPORT downstream (rasters + PDFs) — only after gates pass
python brand/tools/export_png.py            # specimen/mockup PNGs -> exports/png, exports/mockups
```

Stages 0→4 are **gating** (build fails). 5–6 are **reporting** (print findings; US-16 reads them). 7 runs only on green. `render_diff.py` already has `--svg` (cairosvg+Chromium of one file) and `--max-blob`; US-13 adds a `--gate <dir>` mode that loops every `dist/outlined/*.svg`, also diffs against the matching live master, and exits nonzero on any blob >150 (the spike already provides `structural_diff`, `ink_mask`, `render_both`).

### 6. Sources vs generated artifacts

| Kind | Files | Hand-edited? | Committed? |
|---|---|---|---|
| **Source** | `brand/spec/brand.json`, `brand.schema.json`, `templates/*.svg.j2`, `macros.svg.j2`, `tools/*.py`, `fonts/**`, `palette/colors.json`, `guidelines/*.md` | yes | yes |
| **Generated — live-text masters** | the 21 SVGs in `logos/ icons/ lockups/ seal/` | **no** (regenerated) | yes |
| **Generated — outlined masters** | `dist/outlined/*.svg` | no | yes (distribution) |
| **Generated — rasters/PDF** | `exports/png/*`, `exports/mockups/*`, `exports/pdf/*` | no | yes |
| **Scratch** | `tools/_scratch/**` | no | **no** (gitignored) |

A header comment is injected by `generate.py` into every generated SVG immediately after `<svg …>`: `<!-- GENERATED from brand/spec/brand.json by brand/tools/generate.py — DO NOT EDIT. Run build.sh. -->`. This is the loud-fail guard against silent hand-edits.

### 7. Exports policy

- **Committed:** outlined masters (`dist/outlined/`) are the distribution deliverable consumers embed; the live-text masters stay committed (current consumers/specimens reference them, and they are the human-readable form). PNG/PDF/mockup exports stay committed because the repo already ships them (`exports/png` = 10 specimens, `exports/pdf` = 2 books, `exports/mockups` = 9 mockups) and US-9 built tooling for them.
- **Regenerated, not authored:** every export is downstream of stage 7; no export is ever hand-edited. If an export and its source disagree, the export is wrong and `build.sh` overwrites it.
- **Not committed:** everything under `tools/_scratch/` (already gitignored): geometry.json, render PNGs, fallback renders, tmp SVGs.

## Residual risk

- **`text-anchor` outlining is the one un-spiked path.** All 10 icons and the centered lockup runs depend on it. Mitigation: US-13 must add it to `outline.py` and the spike-style gate must cover at least one `middle`-anchored asset before migrating icons (migration step 2). If it fails the 150 gate, the live-text master remains canonical for that asset and only the outlined master is withheld — the build degrades, it does not break.
- **Inter 300 not vendored** — the donation supporting line cannot be outlined at its true weight. Low impact (one decorative caption at fs=26, stone-gray); documented above. Fix: vendor `inter-300.ttf` in US-13 if the 400 substitute is visibly heavier.
- **Bindu cy- and r-ratio inconsistency across existing assets** (cy: wordmark 0.233, R-monogram 0.170, R-circle 0.171, favicon 0.212; r: 0.083 / 0.070 / 0.065 / 0.085). ## Durable or provisional

- **Durable:** the entire architecture — `brand.json` single-source model, the 8-template/macro structure, the `outline.py` API contract, the `build.sh` stage order (validate→generate→outline→render→gate→measure→contrast→export), the sources-vs-artifacts boundary, the render-fidelity gate at max-blob ≤150 (spike-calibrated), the ratios-as-rules principle (bindu positioned by rule, clear-space as cap-height), the exports policy, and the migration order. None of these depend on Cinzel's specific letterforms.
- **Provisional (tied to Cinzel's shapes):** the *literal* bindu numbers (`cx` ink-center value ≈126.5 for the wordmark R; the family-specific `cy_offset_ratio` 0.233/0.170/0.171/0.212 and `r_ratio` 0.083/0.070/0.065/0.085), the `type_styles` sizes/weights/letter-spacing that were tuned to Cinzel, and the monogram baselines. A Phase-2 letterform replacement re-runs `ink_center_x` and re-tunes these literals **in `brand.json` only** — the machine that consumes them is unchanged. That separation (durable generator, provisional values) is the point of the architecture.
